Linux, junto con el sistema GNU (que está estrechamente asociado con la Free Software Foundation (FSF)), forman en conjunto lo que se conoce como GNU/Linux o, a veces, GNU Linux, constituyendo un sistema operativo completo que es ampliamente utilizado hoy en día por millones de personas en todo el mundo.
¿Por qué se llama "disco" si ya no es un disco? Las PC se popularizaron en una época en la que todos los medios de almacenamiento masivo eran discos: teníamos disquetes (floppy disks), discos duros (hard disks) y discos compactos (CDs), todos circulares y planos, como lo que se espera de un disco.
Un disco móvil es un término para un disco portátil, que hoy en día se refiere a un disco duro (HDD) o una unidad de estado sólido (SSD) conectada por USB externamente, sin necesidad de instalarlo dentro del ordenador donde se conectaría vía SATA o PCIe. También es un término que se usa para productos tipo pendrive USB vendidos por TwinMOS.

¿Por qué los SSD son más rápidos que los HDD? Los discos duros (HDD) son más lentos porque los datos se almacenan en un disco que necesita girar para ser leídos, mientras que en las unidades de estado sólido (SSD) los datos se acceden tan rápido como la electricidad.
El disco duro (HDD) gira a una velocidad muy alta, y la aguja tiene una precisión microscópica. Todo eso es impresionante, pero el disco aún necesita girar, lo que genera algo de latencia.
Los SSD almacenan datos atrapando electrones dentro de ellos en algo llamado memoria flash no volátil. Esta tecnología de memoria flash también se utiliza en pendrives, por lo que un SSD es básicamente un pendrive grande instalado dentro del PC en lugar de estar conectado a un puerto USB.

Una placa madre (a veces llamada MoBo) es el componente físico interno más importante de una computadora típica. Es el medio por el cual todos los componentes electrónicos de tu computadora se comunican entre sí. Como el procesador (CPU) está instalado físicamente en un lugar y la memoria RAM en otro, para que el CPU pueda comunicarse con la RAM (y viceversa), debe haber un camino entre ambos. Ese camino está grabado en la placa madre.
La placa madre es un conjunto de caminos por donde viajan los datos. Es un conjunto de circuitos electrónicos por los que pasan señales electrónicas. Si los datos fueran autos, la placa madre sería una ciudad llena de calles.
Diferencia entre CPU y GPU: Ambos son componentes importantes de los sistemas informáticos, pero están asignados a tareas diferentes. La CPU tiene pocos núcleos poderosos diseñados para procesamiento general y secuencial. La GPU tiene muchos núcleos pequeños diseñados especialmente para tareas como renderizado gráfico y cálculo en paralelo.
Una Unidad Central de Procesamiento (CPU) es comúnmente conocida como el cerebro de la computadora. Es un procesador convencional o general que se usa para una amplia gama de operaciones, desde instrucciones del sistema hasta programas.
Los procesadores Intel son los más eficientes para operaciones de un solo hilo, como escribir o navegar por internet.
Las CPUs son capaces de realizar diversas tareas, incluyendo sistemas operativos y software de aplicaciones.
Las CPUs pueden ejecutar varios programas al mismo tiempo gracias a la asignación de hilos a distintas tareas.
La Unidad de Procesamiento Gráfico (GPU) está diseñada para el procesamiento en paralelo y utiliza memoria dedicada conocida como VRAM (memoria de video). Están diseñadas para manejar miles de operaciones al mismo tiempo para tareas como renderizado de imágenes, gráficos 3D, procesamiento de video y ejecución de modelos de aprendizaje automático (machine learning). Tiene su propia memoria separada de la RAM del sistema, lo que les permite manejar tareas complejas de alto rendimiento como renderizado y procesamiento de IA de manera eficiente.
API son las siglas de Interfaz de Programación de Aplicaciones, que es un conjunto de protocolos de comunicación y subrutinas utilizadas por varios programas para comunicarse entre sí.

Decimos que un programa falló en silencio, se cerró en silencio o hizo algo en silencio si lo hizo sin mostrar mensajes al usuario (ni errores ni confirmaciones). Hacer cosas en silencio es un problema de usabilidad, ya que impide que el usuario sepa qué pasó y puede llevarlo a repetir acciones destructivas.
Una fuga de memoria (memory leak) es un error de programación donde el programa consume cada vez más memoria porque nunca libera la que ya no necesita. Al final, se queda sin RAM y el sistema no puede seguir funcionando.
Diferentes lenguajes de programación manejan la reserva de memoria de distintas formas, pero todos necesitan usar RAM. Aunque teóricamente se puede hacer cálculos sin RAM usando solo registros del CPU, en la práctica todo programa guarda datos en RAM.
Las fugas de memoria suceden principalmente en lenguajes de bajo nivel como C y C++, donde el programador debe reservar y liberar memoria manualmente. Lenguajes de alto nivel como Python o Java tienen recolectores de basura (GC) que liberan la memoria automáticamente.
Aun en lenguajes con recolección de basura como JavaScript se pueden generar fugas de memoria. Basta con crear un objeto que no se borra cuando ya no se necesita, y así sigue ocupando memoria.
"Borrar" significa eliminar datos, pero no exactamente. Borrar archivos, carpetas o capas en un editor no reduce la memoria del equipo, solo cambia el contenido que hay en memoria.
El software de código abierto es aquel cuyo código fuente está disponible públicamente. Generalmente, "código abierto" implica que también puede ser modificado y redistribuido (libre), bajo licencias como GPL, LGPL, MIT, etc. Si el código está disponible pero no se puede modificar ni redistribuir, se le dice “source available”.
En Linux, un daemon es un programa que corre en segundo plano todo el tiempo, prestando servicios a otros programas o esperando peticiones de la red. Son equivalentes a los “servicios” en Windows. Por convención, sus nombres terminan en “d”, como systemd, httpd, mysqld, dockerd, etc.
